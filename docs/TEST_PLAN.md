## Affected Pages/Routes
- `adk web` (local demo UI) — the only surface; exercise the full agent conversation here.

## Key Interactions to Verify
- Upload model-plate photo + symptom -> agent reads brand / model / error code.
- Clarify-then-diagnose: agent asks >=1 clarifying question, then recommends ranked safe fixes.
- Reopen the case (HEADLINE): fresh session, supply `case_id`, agent recaps prior steps and continues.
- Escalation: after fixes are exhausted, agent DRAFTS (never sends) an email with model + steps + contact.

## Edge Cases
- Blurry/unreadable plate -> retake prompt (cap 2) then type-in.
- Valid-but-wrong model number (transposition) -> membership re-prompt.
- Dangerous request (gas/electrical/water/refrigerant) -> deterministic refusal + escalate.
- iFixit/grounding unavailable -> falls back to curated table seamlessly.
- Reopen unknown/corrupt `case_id` -> clear error, no crash.
- Gemini throttled -> recorded-response fixture keeps the demo moving.

## Critical Paths
- Photo -> diagnosis -> fix -> resolved (happy path).
- Reopen-the-case: close, reopen by `case_id`, recap, continue (the headline).
- Safety refusal end to end (the Day-4 artifact).

## Eval suites to run (LLM) — run all three before recording
- `diagnosis_eval` — 10 symptoms, >=16/20 first-fix AND asks >=1 clarifying question.
- `plate_read_eval` — N/M model-number reads across ~3 lighting conditions.
- `safety_eval` — 0 unsafe answers.
